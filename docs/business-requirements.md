## 一、用户角色定义

| 角色 | 说明 | 典型用户 |
|------|------|----------|
| **普通用户（买家）** | C 端消费者 | 注册用户、游客 |
| **运营人员** | 后台运营管理 | 商品运营、订单客服、营销运营 |
| **系统管理员** | 系统最高权限 | IT 管理员、DevOps |

## 五、企业级非功能需求

### 5.1 安全需求

| 编号 | 需求 | 说明 |
|------|------|------|
| SEC-01 | 认证安全 | JWT 令牌 + 刷新令牌，令牌有效期控制 |
| SEC-02 | 密码安全 | BCrypt 加密，密码强度校验（8 位以上 + 字母数字特殊字符） |
| SEC-03 | 接口安全 | 参数校验（防止 SQL 注入、XSS）、敏感接口限流 |
| SEC-04 | 传输安全 | HTTPS 全站加密（生产环境） |
| SEC-05 | 权限安全 | RBAC 模型，最小权限原则，API 级别权限控制 |
| SEC-06 | 数据安全 | 敏感数据脱敏（手机号、邮箱部分掩码）、日志不打印敏感信息 |
| SEC-07 | CSRF 防护 | Token 方式防止跨站请求伪造 |
| SEC-08 | 验证码 | 登录/注册/敏感操作图形验证码或滑块验证 |
| SEC-09 | 会话管理 | Token 过期策略、单设备登录（可选） |
| SEC-10 | 审计追踪 | 所有管理员操作可追溯 |

### 5.2 性能需求

| 编号 | 需求 | 说明 | 目标值 |
|------|------|------|--------|
| PERF-01 | 接口响应时间 | 核心接口 P99 延迟 | < 200ms |
| PERF-02 | 页面加载时间 | 首屏加载 | < 2s |
| PERF-03 | 并发支持 | 商品浏览 QPS | > 5000 |
| PERF-04 | 下单 TPS | 峰值下单吞吐 | > 500 TPS |
| PERF-05 | 数据库查询 | 单条查询 | < 50ms |
| PERF-06 | 缓存命中率 | 热点数据 | > 95% |
| PERF-07 | 消息延迟 | 订单超时等延迟场景 | < 5s |

### 5.3 可观测性需求

| 编号 | 需求 | 说明 |
|------|------|------|
| OBS-01 | 健康检查 | 应用健康检查端点（/actuator/health） |
| OBS-02 | 日志规范 | 结构化日志（JSON 格式）、链路追踪 ID（TraceId）、日志级别动态调整 |
| OBS-03 | 指标监控 | JVM 指标、接口 QPS/RT/错误率、数据库连接池、Redis 连接、消息队列堆积 |
| OBS-04 | 告警 | 错误率 > 1% 告警、接口 RT > 500ms 告警、数据库慢查询告警 |
| OBS-05 | 链路追踪 | 全链路 TraceId 传递（后期引入 SkyWalking/Jaeger） |

### 5.4 数据一致性需求

| 编号 | 需求 | 说明 | 方案 |
|------|------|------|------|
| CON-01 | 库存扣减一致性 | 下单扣库存不能超卖 | 乐观锁 + 数据库唯一约束 |
| CON-02 | 订单支付一致性 | 支付成功必须更新订单状态 | 支付回调 + 主动查询 + 幂等 |
| CON-03 | 订单超时一致性 | 超时取消必须释放库存+优惠券 | RabbitMQ 延迟消息 + 幂等处理 |
| CON-04 | 退款一致性 | 退款成功必须恢复库存 | 事件驱动 + 补偿机制 |
| CON-05 | 最终一致性 | 非核心链路（通知、日志） | 本地消息表 + 消息重试 |

### 5.5 可用性需求

| 编号 | 需求 | 说明 |
|------|------|------|
| AVL-01 | 系统可用性 | 99.9%（年停机 < 8.76 小时） |
| AVL-02 | 故障恢复 | 数据库/Redis/RabbitMQ 自动重连 |
| AVL-03 | 优雅关闭 | Spring Boot 优雅关闭，不丢失处理中的请求 |
| AVL-04 | 限流降级 | 核心接口限流，非核心接口熔断降级（后期引入 Sentinel） |

### 5.6 扩展性需求

| 编号 | 需求 | 说明 |
|------|------|------|
| EXT-01 | 水平扩展 | 应用无状态，支持多实例部署 |
| EXT-02 | 模块拆分 | 当前模块化单体，未来按领域拆分为微服务 |
| EXT-03 | 数据库扩展 | 读写分离预留（后期）、分库分表预留 |
| EXT-04 | 多租户 | 暂不考虑，但数据隔离方案预留 |

## 六、功能优先级总览

```
P0（MVP，阶段 0-8）：
  用户注册登录 → 商品浏览 → 购物车 → 下单 → 支付 → 订单管理
  后台：商品管理、库存管理、订单管理、用户管理、权限管理

P1（第二阶段，阶段 9-12）：
  营销活动、售后、评价、数据统计、经营报表

P2（第三阶段，阶段 13-15）：
  秒杀、推荐、IM 客服、第三方登录、多租户
```
